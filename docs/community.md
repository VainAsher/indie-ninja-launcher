# Community Guide

This game is built in public, and community feedback is part of the development loop.

---

## How You Can Help

You do not need technical skills to make useful contributions.

High-value ways to help:

- report a bug with simple repro steps
- share a confusing moment in UI or controls
- call out a mission or mechanic that felt especially good
- test host/join flow with a friend and report what broke

---

## Best Report Format

Use this lightweight format in launcher Report or GitHub issue:

1. Version tag (`v0.x` game, `v1.x` launcher if relevant)
2. Mode (`Campaign`, `Developer`, `Arcade`, `Host`, `Join`)
3. Steps you took
4. What you expected
5. What happened

Optional but very useful:

- short log snippet from `user_data/logs/`
- screenshot or short clip

---

## Weekly Community Habit

If you want to contribute regularly, this cadence works well:

- play 1-2 short sessions each week
- submit at least one issue or feedback note
- check the changelog for what got fixed or adjusted
- re-test one old issue after a new release

---

## Feedback Scope

Most useful now:

- onboarding clarity
- control readability
- mission pacing
- multiplayer connection reliability
- crashes and softlocks

---

## Where To Submit

- Launcher Report tab (recommended)
- Direct issue:
